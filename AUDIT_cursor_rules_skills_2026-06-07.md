### 5 highest-impact issues

1. **"Five locked parity hashes" is wrong everywhere.** The skill family universally claims "five" while every table that lists them shows six rows (L1, L2, L3, L3-orders, L4, L5). The actual `tests/determinism/parity_manifest.py` registers **eleven** locked baselines, and the test file names cited in the tables (`test_sensor_replay.py`, `test_hazard_parity.py`) **do not exist** â€” the real files are `test_sensor_reading_replay.py` and `test_regime_hazard_replay.py`. An agent following any of the four skills that "own" or cite parity-hash baselines will look at the wrong test files and assert the wrong count. (Finding IDs F-01, F-02, F-03.)
2. **`SensorProtocol` in `feature-engine/SKILL.md` does not match the implementation.** SKILL.md claims `class SensorProtocol(Protocol)` with `update(event) -> SensorReading | None`, `is_warm(symbol)`, and `reset(symbol)`. The actual `src/feelies/sensors/protocol.py` defines `class Sensor(Protocol)` with `update(event, state, params)`, `initial_state()`, **and explicitly says "**No** `is_warm` method"** in its docstring. The `SensorSpec` field names are also different (`subscribes_to`/`input_sensor_ids`/`min_history`/`throttled_ms` in code vs. `depends_on`/`warm_up`/`throttle_ns` in the skill). Anyone authoring or extending a sensor against this skill will write code that fails type-checking. (Findings F-08, F-09.)
3. **Garbled glossary text + self-contradictory history in `platform-invariants.mdc`.** Line 52 (`feature` entry) contains the malformed fragment `â€¦bit-identical.ost-PR-2b-i.` (missing the "p" in "post"), and the same entry describes PR-2b-iv as both "(D.2 COMPLETE) â€” deleted the remaining test scaffolding" **and** "A future PR-2b-iv will wire `SizedPositionIntent` â†’ `OrderRequest`". Since this rule has `alwaysApply: true`, the contradiction is always in context. (Findings F-04, F-05.)
4. **`DataHealth` SM is documented as 3 states everywhere; the code has 4** (HEALTHY, GAP_DETECTED, **HALTED**, CORRUPTED) â€” see `src/feelies/ingestion/data_integrity.py:17-52`. Skills tell agents the SM is "HEALTHY â†” GAP_DETECTED; CORRUPTED terminal" and call this "three states", missing the HALTED state and its transitions. (Finding F-10.)
5. **Multiple skills reference sensors that do not exist in `src/feelies/sensors/impl/`** â€” `hawkes_intensity_imbalance` (microstructure-alpha SKILL.md), `kyle_lambda_300s`, `trade_clustering`, `micro_price_drift`, `effective_spread`, `quote_replenishment_asym` (microstructure-alpha/system-architecture.md). The microstructure-alpha SKILL.md itself warns against four of these on line 119 â€” but its sibling system-architecture.md still lists them in the canonical catalog. (Findings F-06, F-07.)
