## Decision-economics audit

**RC-A status.** The legacy translator is still economically blind: it chooses from the 7-intent matrix using signal direction and `position.quantity` only (`src/feelies/execution/intent.py:73`, `src/feelies/execution/intent.py:103`). It does not read avg entry, unrealized PnL, age, spread, or cost. Same-direction over-target LONG and SHORT both return `NO_ACTION` (`src/feelies/execution/intent.py:150`, `src/feelies/execution/intent.py:187`).

**G-1 equivalence and current default.** `LegacyPositionManager` intentionally reproduces the translator: over-target same-direction books clamp to the current side and never emit TRIM (`src/feelies/execution/position_manager.py:207`, `src/feelies/execution/position_manager.py:238`, `src/feelies/execution/position_manager.py:279`). That equivalence is tested across translator/manager cases (`tests/execution/test_position_manager.py:48`). In this checkout, however, the boot path wires a `TargetPositionManager` and drives it from `PlatformConfig` defaults (`src/feelies/bootstrap.py:553`, `src/feelies/bootstrap.py:725`, `tests/bootstrap/test_position_manager_wiring.py:142`). That is an intentional default-on trade-path choice, not a hidden default-off feature.

**Trim path.** A same-direction target shrink has landed as `PlanLeg.TRIM` in `TargetPositionManager`: it only overrides the legacy hold for same-direction shrink, applies a churn threshold, optionally suppresses the trim if forward edge still clears trim round-trip cost, then emits a partial reducing order (`src/feelies/execution/position_manager.py:391`, `src/feelies/execution/position_manager.py:399`, `src/feelies/execution/position_manager.py:440`, `src/feelies/execution/position_manager.py:480`). There is no separate `TradingIntent.SCALE_DOWN`; the adapter maps TRIM to a partial `EXIT` so it inherits exit bypass semantics (`src/feelies/execution/position_manager.py:682`).

**B4 arithmetic.** B4 is off only when the ratio is <= 0 or no cost model is wired (`src/feelies/kernel/orchestrator.py:3117`). Otherwise it computes round-trip cost (`src/feelies/kernel/orchestrator.py:3119`) and compares against the realization-calibrated edge (`src/feelies/kernel/orchestrator.py:3130`). The cost estimator used by B4 has a depth-aware aggressive leg and explicitly states it mirrors market-fill behavior (`src/feelies/execution/cost_model.py:459`, `src/feelies/execution/cost_model.py:476`, `src/feelies/execution/cost_model.py:570`).

**B5 arithmetic.** B5 prices reversal as exit round-trip cost plus entry round-trip cost, then compares raw `edge_estimate_bps` against `multiplier x combined_cost` (`src/feelies/kernel/orchestrator.py:3240`, `src/feelies/kernel/orchestrator.py:3263`, `src/feelies/kernel/orchestrator.py:3281`). Under the same raw-edge arithmetic, a reversal that passes B5 cannot have the exit component alone exceed the combined required threshold because the exit component is included in the sum. The residual issue is consistency: B5 uses raw edge (`src/feelies/kernel/orchestrator.py:4361`) while B4 later uses calibrated edge (`src/feelies/kernel/orchestrator.py:3130`, `src/feelies/kernel/orchestrator.py:4409`). If B5 passes but B4 fails, the system intentionally submits a flatten-only reversal. That is fail-safe, but the B5 alert/trace may overstate the economics relative to calibrated B4.

## Sizing audit

**Base sizer.** `BudgetBasedSizer` computes `floor(equity x allocation% x strength x regime_factor / price)` and caps at max shares (`src/feelies/risk/position_sizer.py:47`, `src/feelies/risk/position_sizer.py:90`, `src/feelies/risk/position_sizer.py:106`). It returns zero for non-positive price/equity (`src/feelies/risk/position_sizer.py:87`), clamps strength to `[0, 1]` (`src/feelies/risk/position_sizer.py:94`), and clamps regime EV at `min(1.0, ev)` so regime cannot amplify exposure (`src/feelies/risk/position_sizer.py:130`).

**Tilt math.** `SizerTiltConfig` disables edge, vol, and inventory factors by default (`src/feelies/risk/edge_weighted_sizer.py:44`). Edge tilt is `edge / ref` clamped to `[floor, cap]` and no-ops on non-positive reference (`src/feelies/risk/edge_weighted_sizer.py:118`); vol tilt no-ops on missing/non-positive realized vol (`src/feelies/risk/edge_weighted_sizer.py:130`); inventory taper never amplifies (`src/feelies/risk/edge_weighted_sizer.py:144`). FLAT signals always get unit tilt so exits are never shrunk by G-7 (`src/feelies/risk/edge_weighted_sizer.py:195`, `src/feelies/risk/edge_weighted_sizer.py:207`). `apply_tilt` floors deterministically and re-caps at `max_position` (`src/feelies/risk/edge_weighted_sizer.py:282`).

**Shadow discipline.** Bootstrap builds both base and tilted sizers but chooses the tilted sizer only when `sizer_tilt_drive` is true (`src/feelies/bootstrap.py:525`, `src/feelies/bootstrap.py:551`). Defaults keep the base sizer live and all tilt factors off (`src/feelies/core/platform_config.py:374`, `tests/bootstrap/test_position_manager_wiring.py:155`). Explicit opt-in routes live sizing through `EdgeWeightedSizer` (`tests/bootstrap/test_position_manager_wiring.py:165`).

## Netting audit

**Standing target expiry.** `StandingTarget` remains fresh at `now_ns == expiry_ns`, matching the signal-buffer fresh-at-boundary policy (`src/feelies/execution/portfolio_netter.py:30`, `src/feelies/execution/portfolio_netter.py:106`). Targets use `signal_ts + k x horizon_seconds`; non-positive horizon or `k` means no expiry (`src/feelies/execution/portfolio_netter.py:52`). The orchestrator explicitly evicts horizon-zero net-shadow targets on the next tick so transient signals do not linger (`src/feelies/kernel/orchestrator.py:4145`).

**Default inertness.** `enable_portfolio_netting` defaults false (`src/feelies/core/platform_config.py:354`). With no sink and netting disabled, `_record_net_shadow` returns immediately (`src/feelies/kernel/orchestrator.py:4125`). PORTFOLIO target shadowing is measurement-only and disabled when live netting drives, avoiding double-counting (`src/feelies/kernel/orchestrator.py:4067`, `src/feelies/kernel/orchestrator.py:4073`).

**Live netting behavior.** When enabled, portfolio netting collapses live standing targets into one signed desired target by clamping per-alpha targets, summing them, applying a portfolio cap, and computing weighted edge/urgency from contributors aligned with the net direction (`src/feelies/execution/portfolio_netter.py:152`, `src/feelies/execution/portfolio_netter.py:161`, `src/feelies/execution/portfolio_netter.py:174`). Forced-market-exit strategies bypass the net target so stop/session safety exits always target flat (`src/feelies/kernel/orchestrator.py:2474`, `src/feelies/kernel/orchestrator.py:2480`).

**Determinism.** Live targets are sorted by `strategy_id` before netting (`src/feelies/execution/portfolio_netter.py:125`), and tests cover order-independent netting (`tests/execution/test_portfolio_netter.py:110`). The only set-returning helper, `DesiredTargetBook.symbols`, is not in the live `net()` decision path (`src/feelies/execution/portfolio_netter.py:136`).

## PnL-ledger audit

**Avg-cost store lifecycle.** The parity-bearing store opens at fill price when flat, blends avg entry on same-sign adds, realizes PnL on reductions, resets avg on cross-through-zero residual and on flat (`src/feelies/portfolio/memory_position_store.py:70`, `src/feelies/portfolio/memory_position_store.py:75`, `src/feelies/portfolio/memory_position_store.py:83`, `src/feelies/portfolio/memory_position_store.py:88`). Example: buy 100 @ 10, buy 100 @ 12 gives avg 11; sell 250 @ 13 realizes `(13 - 11) x 200 = 400` and opens short 50 @ 13; buy 50 @ 9 realizes `(13 - 9) x 50 = 200`; total realized 600.

**Lot ledger reconciliation.** FIFO lots are explicitly observability-only and not the parity-bearing book (`src/feelies/portfolio/lot_ledger.py:1`, `src/feelies/portfolio/lot_ledger.py:14`). The same example realizes FIFO 300 + 100 on the long close and 200 on the short cover, also total 600. Tests pin equality at flat over deterministic and randomized streams (`tests/portfolio/test_lot_ledger.py:111`, `tests/portfolio/test_lot_ledger.py:137`), while the ledger doc correctly allows divergence during partial reduces (`src/feelies/portfolio/lot_ledger.py:16`).

**Marks.** Longs mark to bid, shorts to ask, with mid fallback (`src/feelies/portfolio/memory_position_store.py:97`, `src/feelies/portfolio/memory_position_store.py:138`). Unrealized PnL uses `(mark - avg) x quantity`, which is sign-correct for shorts because quantity is negative (`src/feelies/portfolio/memory_position_store.py:145`).

**Strategy aggregate.** `StrategyPositionStore.get_aggregate` sums realized/unrealized/fees across strategy stores and computes net avg entry from signed cost over net quantity (`src/feelies/portfolio/strategy_position_store.py:95`, `src/feelies/portfolio/strategy_position_store.py:111`, `src/feelies/portfolio/strategy_position_store.py:120`). The doc warns mixed directions can produce an aggregate avg that is lower or negative; this is a net-book view, not per-alpha cost basis (`src/feelies/portfolio/strategy_position_store.py:98`).

**Trade journal.** The journal records order id, symbol, strategy, side, quantities, fill price, timestamps, cost, fees, realized delta, correlation id, and trading intent (`src/feelies/kernel/orchestrator.py:5756`). It does not propagate `order.reason`, `order.source_layer`, or other metadata despite the schema having `metadata` (`src/feelies/storage/trade_journal.py:49`). That is not a ledger-math bug, but it weakens Inv-13 for post-trade forensics.

**Documentation inconsistency.** `TradeRecord.net_pnl` still says spread is recorded in fees and realized PnL is mid-to-mid (`src/feelies/storage/trade_journal.py:52`), while the fill and position-store docs say cross-price spread is embedded in fill price and fees exclude spread (`src/feelies/execution/market_fill.py:219`, `src/feelies/portfolio/position_store.py:22`). The implementation `realized_pnl - fees` is still the right net formula; the doc is stale.
